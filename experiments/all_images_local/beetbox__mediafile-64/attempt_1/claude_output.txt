## Summary

The issue has been successfully fixed! The problem was that list fields (like `genres`, `artists`, `images`, etc.) were returning an empty list `[]` when the tag didn't exist, while non-list fields correctly returned `None`. This inconsistency has been resolved.

### Changes Made

**In `/testbed/mediafile.py`:**

1. **Updated `ListStorageStyle.fetch()`** - Changed from returning `[]` to `None` when tag doesn't exist (line 637)

2. **Updated `ListStorageStyle.get_list()`** - Added None check before deserializing (lines 625-629)

3. **Updated `ListStorageStyle.get()`** - Added None check before accessing list element (lines 613-620)

4. **Updated `ListStorageStyle.set()`** - Added None handling when setting single value (lines 643-646)

5. **Updated `ListStorageStyle.set_list()`** - Added None handling when setting list values (lines 652-657)

6. **Updated `ListMediaField.__get__()`** - Changed return from `[]` to `None` when no values found (line 1336)

7. **Updated docstring** - Updated to reflect that `fetch()` can return `None` (line 603)

**In `/testbed/test/test_mediafile.py`:**
