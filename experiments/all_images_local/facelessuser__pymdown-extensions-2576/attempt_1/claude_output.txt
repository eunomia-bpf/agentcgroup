## Summary

Fixed the FancyLists `KeyError` issue when list items have only a separator without a number (e.g., `. bar` or `) bar`).

**Root Cause:** When the separator (`. ` or `)`) appears without a preceding number, the `value` is an empty string, causing `list_type` to be just `'dot-'` or `'paren-'`, which doesn't exist in the `OL_TYPES` dictionary.

**Fix:** Added a check for empty `value` in `get_fancy_type` method (pymdownx/fancylists.py:303-305) to default to decimal type.

**Testing:** 
- All 48 fancylists tests pass
- The fix correctly handles both examples from the issue
- The failing test in test_captions.py is a pre-existing issue unrelated to this fix
